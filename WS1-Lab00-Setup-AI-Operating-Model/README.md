# Lab 0: Setup + AI Operating Model

## Objective
Standardize how the team uses Copilot in SSMS so outputs are consistent, auditable, and production-relevant.

## Tools
- SSMS 22.7.0
- Copilot Chat in SSMS

## AI-first flow (majority AI work)
1. AI/chat work (majority):
   - Run Prompt Drill #1 and Prompt Drill #2 in Copilot Chat.
   - Use at least one refinement turn to compare readability-first and performance-first output.
2. Manual execution work (supporting validation):
   - Execute the standalone query script and run validation SQL generated by AI.

## Steps
1. Confirm Copilot chat opens inside SSMS.
2. Confirm model picker shows locally validated models.
3. Set reasoning level based on task complexity:
   - Low: small rewrites, quick fixes
   - Medium: standard query/procedure generation
   - High: architecture-level SQL design, complex tuning
4. Practice context scoping:
   - `#selection` for focused edits
   - `#editor` for full active query
   - `#file` for referenced SQL file
5. Run slash commands baseline:
   - `/explain`
   - `/fix`
   - `/tests`
   - `/doc`

## Prompt Drills
- Prompt Drill #1: "Review this query for correctness and performance in an IIoT workload with noisy telemetry. Return findings and a rewritten query."
- Prompt Drill #2: "Generate a reusable checklist for SQL proc changes that includes plan review, row correctness checks, and test updates."
- Prompt Drill #3: "Given this alarm rollup query, propose two alternatives: one for readability and one for performance."

## How to perform the key steps
1. Open `scripts/01_lab0_standalone_query.sql` in SSMS and execute it once.
2. Keep the query selected in SSMS and run this prompt in Copilot chat:
   - "Review this query for correctness and performance in an IIoT workload with noisy telemetry. Return findings and a rewritten query."
3. Run the follow-up refinement prompt:
   - "Now return two rewrites: readability-first and performance-first. Include quick validation SQL for row parity."
4. Compare both rewrites, choose one for the team standard, and keep the validation SQL.
5. Save the final reusable prompt in `prompts/team-prompt-playbook-template.txt`.

## Prompt Playbook (when and how to fill)
1. Before Prompt Drill #1, fill section 1) Query Design Prompt in `prompts/team-prompt-playbook-template.txt`.
2. Before the follow-up rewrite request, fill section 2) Query Tuning Prompt.
3. After the final Copilot response, replace draft wording with the exact final prompt text that worked.
4. In Validation Step, record the validation SQL used and the outcome (parity passed, runtime improved, or issue found).

## Deliverables
- Completed prompt playbook file in `prompts/team-prompt-playbook-template.txt`
- One saved reusable prompt per participant
- One model-routing guideline agreed by team

## Validation
- Every participant can execute one prompt and one follow-up refinement.
- Team agrees on a default model-routing policy by task type.
